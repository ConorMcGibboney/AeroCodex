# Session Decisions Captured in Phase 0.001

## Project identity

Name: **AeroCodex**

Formal title: **AeroCodex: Verified Aerospace Engineering Mathematics in Pure Rust**

Tagline: **The living Rust codex for source-traceable aerospace engineering math.**

## Core philosophy

- 100% Rust verified core.
- No hidden wrappers around C, C++, Fortran, Python, MATLAB, REFPROP, CEA, Cantera, OpenFOAM, SU2, SPICE, or similar tools.
- Unit-safe public APIs.
- Evidence cards for stable calculations.
- Every public calculation should expose assumptions, source, validity, warnings, evidence, and numerical behavior.
- Certification caveat must remain visible.
- Agents may orchestrate; Rust computes.

## Version decision

The project remains at **0.001** until basic Rust equations exist across broad aerospace categories, bio-regenerative life support, and celestial mechanics / astrodynamics.

Cargo packages use **0.0.1** for SemVer compatibility.

## Deploy decision

The deployment agent should deploy a clean repository root from the new `AeroCodex_repository_foundation_v0_001.zip`, not the older `v0_0_1` zip.

## Agentic decision

The agentic pack is integrated as scaffolding and documentation. It is not yet a production agent server. The first agentic target remains one complete trust-loop vertical slice.
